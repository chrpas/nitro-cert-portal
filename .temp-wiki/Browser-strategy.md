Deciding which browsers to support is a balance between on one hand giving users with older browsers an acceptable experience and on the other hand ease of o development, maintenance, and user experience for users with newer browser.

Browsers today auto update to a large degree which makes version support less of a problem than in the past. Safari is the outlier here by using yearly updates tied to iOS version.

Our support is based on offering the best experience to the browser versions covered in a global usage of > 0.5%. This is the default filter set on caniuse.com, a site for checking browser support for specific features. This browser support is illustrated in the chart below from caniuse.com with current statistics.

In addition to this we have functional support* for Safari two major versions back. Meaning if current version is `17.x` then things should work in at least version `15.x`.

(*) Things are useful rather than decorative.
