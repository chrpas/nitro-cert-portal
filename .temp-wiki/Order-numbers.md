## Prefixes
In startersite, the order numbers are prefixed with environment specific prefixes, to prevent colliding order numbers between environments. In prod, order numbers are prefixed with "O", in preprod with "PREP" and on integration with "INT". Locally is uses your username.

The prefixes can be customized in `appsettings.json`. You can specify your own local prefix in `appsettings.Development.json` if you want e.g. your initials instead of your full username:

```js
"Nitro": {
    "Order": {
      "OrderNumberPrefix": "MG"
    },
```


> There is also the `PaymentProviderOrderNumberPrefix` setting but this is not used on Epi order numbers. Instead it is used by some of our payment packages (currently Vipps, Stripe and NetsEasy), and is used when creating the payment at the PSP to reference the epi order. So e.g. order O123 in nitro would be PREFIXO123 in Vipps.  

## Resetting the sequence
If you need to reset order numbers, you can reset the `dbo.OrderNumberSequence` by using the following SQL command:

`ALTER SEQUENCE OrderNumberSequence RESTART WITH <number>`

More general information for SQL sequences can be found here: https://learn.microsoft.com/en-us/sql/t-sql/statements/alter-sequence-transact-sql
